# Wellness Architect — WordPress 테마

React/Vite로 만든 "웰니스 아키텍트(오승우)" 롱제비티 랜딩 페이지를 **빌드 과정 없이 바로 사용 가능한 클래식 WordPress 테마**로 이식한 버전입니다.

## 설치 방법

### 방법 A — ZIP 업로드 (가장 간단)
1. `wellness-architect.zip` 파일을 준비합니다. (같은 폴더에 생성되어 있음)
2. WordPress 관리자 → **외모(Appearance) → 테마(Themes) → 새로 추가 → 테마 업로드**
3. zip 파일 선택 후 **지금 설치 → 활성화**

### 방법 B — 폴더 직접 복사 (FTP/호스팅 파일 관리자)
1. `wellness-architect` 폴더 전체를 서버의 `wp-content/themes/` 안에 업로드
2. 관리자 → 외모 → 테마에서 **Wellness Architect** 활성화

## 활성화 후 필수 설정

1. **홈페이지를 이 랜딩으로 지정**
   - 관리자 → **설정(Settings) → 읽기(Reading)**
   - "홈페이지 표시"를 **정적인 페이지**로 선택하거나, 기본 최신 글 상태로 두어도 `front-page.php`가 자동으로 랜딩 페이지를 표시합니다.

2. **사이트 정보**
   - 외모 → **사용자 정의하기(Customize) → Wellness Architect 설정** 에서 아래 항목을 편집할 수 있습니다.
     - 상담 신청 수신 이메일
     - 히어로 제목 / 부제
     - 인스타그램 URL / 네이버 카페 URL
   - 사이트 제목/태그라인(헤더 로고 텍스트)은 **설정 → 일반**에서 변경합니다.

3. **상담 폼 메일 수신**
   - 1:1 컨설팅 폼은 `wp_mail()`로 발송됩니다. 메일이 안 오면 **WP Mail SMTP** 같은 SMTP 플러그인을 설치하세요(공유 호스팅 권장).

## 관리자에서 콘텐츠 편집 (ACF)

이 테마는 **Advanced Custom Fields(ACF)** 로 모든 섹션을 관리자에서 직접 편집할 수 있습니다.

1. **ACF PRO 플러그인 설치·활성화**
   - 옵션 페이지 + 반복 필드(Repeater)를 사용하므로 **ACF PRO**가 필요합니다.
   - 미설치 상태에서도 테마는 **기본 콘텐츠로 정상 동작**합니다(편집만 불가). ACF가 없으면 관리자에 안내 알림이 표시됩니다.

2. **편집 위치**: 관리자 → **외모(Appearance) → 테마 콘텐츠**
   - 상단 탭으로 섹션이 나뉘어 있습니다: `헤더/메뉴 · 히어로 · 소개 · 4대 기둥 · 청춘리셋 · 바이오해킹 툴 · 상담 · 푸터`
   - 필드는 코드로 자동 등록되므로 **수동으로 필드를 만들 필요가 없습니다.**

3. **동작 방식 (중요)**
   - 텍스트 필드는 기본값이 미리 채워져 있습니다.
   - 카드/메뉴 같은 **반복 필드는 비워두면 테마 기본 콘텐츠가 그대로 표시**되고, 한 줄이라도 입력하면 입력한 내용으로 **전체가 교체**됩니다. (예: 기둥 카드를 1개만 넣으면 1개만 표시됨 → 편집 시 기본 4개를 모두 복제해 넣고 수정하는 방식 권장)
   - 이미지/영상은 미디어 라이브러리에서 업로드해 교체할 수 있습니다.

4. 상담 수신 이메일은 **상담 탭**에서 지정할 수 있고, 비워두면 사이트 관리자 이메일로 발송됩니다.

> ACF 없이 가볍게 일부만 바꾸려면 **외모 → 사용자 정의하기 → Wellness Architect 설정**에서 수신 이메일·SNS 링크를 편집할 수 있습니다.

## 구조

```
wellness-architect/
├── style.css                  # 테마 헤더 + 전체 스타일(Tailwind 디자인 이식)
├── functions.php              # 테마 설정, 에셋 로드, 메뉴, 폼 처리, SVG 아이콘
├── header.php / footer.php
├── front-page.php             # 랜딩 조립(섹션 호출)
├── index.php / page.php / 404.php
├── inc/
│   ├── content.php            # 콘텐츠 접근자(ACF 우선, 기본값 폴백)
│   └── acf-fields.php         # ACF 옵션 페이지 + 필드 자동 등록
├── template-parts/
│   ├── section-hero.php
│   ├── section-about.php
│   ├── section-pillars.php
│   ├── section-youth-reset.php
│   ├── section-biohacking.php
│   └── section-contact.php
└── assets/
    ├── css가 아닌 images/      # 인물·제품 이미지
    ├── video/hero-video.mp4    # 히어로 배경 영상
    └── js/main.js              # 모바일 메뉴 + 스무스 스크롤
```

## 원본과의 차이 / 참고

- 원본 React의 Tailwind 유틸리티 클래스는 **빌드 의존성을 없애기 위해** 동일한 디자인 토큰 기반의 순수 CSS로 재작성했습니다.
- 콘텐츠(텍스트·제품 목록)는 템플릿 파트의 PHP 배열에 들어 있어 직접 수정하거나 추후 커스텀 필드/ACF로 확장할 수 있습니다.
- 상담 폼은 클라이언트 토스트 대신 서버에서 처리하여 실제 메일 발송 + 성공/오류 메시지를 표시합니다.
- `Ask Ai`, `검색`, `Login` 버튼은 원본과 동일하게 자리만 잡혀 있습니다(필요 시 연결).
